Resumen de Git

¿Qué es Git?

Git es un sistema de control de versiones distribuido que permite a los desarrolladores gestionar el código fuente de sus proyectos de manera eficiente. Permite registrar los cambios realizados en el código, facilitando la colaboración y el seguimiento de versiones.

Comandos Básicos

  • git init: Inicializa un nuevo repositorio de Git.
  • git clone [URL]: Clona un repositorio existente.
  • git status: Muestra el estado actual del repositorio.
  • git add [archivo]: Agrega un archivo al área de preparación.
  • git commit -m "mensaje": Confirma los cambios en el repositorio.
  • git push: Envía los cambios al repositorio remoto.
  • git pull: Obtiene los cambios del repositorio remoto y los integra.

Flujo de Trabajo Básico

  1. Inicializa un repositorio con git init.
  2. Realiza cambios en tu proyecto.
  3. Agrega los archivos modificados con git add.
  4. Confirma los cambios con git commit.
  5. Envía los cambios al repositorio remoto con git push.

Ramas en Git

Las ramas son una característica clave en Git que permite trabajar en distintas versiones de un proyecto al mismo tiempo. El comando git branch se utiliza para listar, crear o eliminar ramas.

  • git branch [nombre-rama]: Crea una nueva rama.
  • git checkout [nombre-rama]: Cambia a la rama especificada.
  • git merge [nombre-rama]: Combina la rama especificada con la actual.
